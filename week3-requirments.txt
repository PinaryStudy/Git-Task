# week3 과제 요구사항입니다.

실습 영상에서 배운 내용을 자유롭게 사용해주세요.

[ 요구 사항 ]
1. 상대 참조 HEAD^ 를 사용하여 reset을 수행한다.
2. reflog를 확인하여 reset을 복구한다.
3. 상대 참조 HEAD~1을 사용하여 reset을 수행한다.
4. revert를 통해 새로운 히스토리를 작성한다. 이 경우 커밋의 제목은 revert: 로 시작하도록 한다.
  4-1. ex -> "revert: B기능 제거"
5. revert 과정에서 conflict가 발생한다면 해결하고 위와 같은 조건으로 commit 한다.

+ 모든 명령어의 사용은 아래 commit 범위에서만 이루어져야 한다.
commit: feat: B기능 추가
commit: feat: A기능 추가
commit: feat: root commit

[ 과제 제출 방법 ]
위 요구 사항을 만족한 후 git 을 사용하는 터미널에서 다음 명령어를 사용합니다.
bash validate-week3.sh

해당 명령어를 사용한 뒤 출력된 내용을 캡쳐하여 노션 과제 제출 방법을 참고하여 Issue로 작성해주세요!
통과 하지 못한 부분이 있는 경우, 과제 제출로 처리하지 않습니다.

또한 +로 명시된 부분의 범위를 넘어가는 경우 과제 제출로 인정하지 않습니다.

[추가]
해당 과제의 경우 주어진 요구사항에서 틀린 경우 validate가 정상적으로 작동하지 않을 수도 있어요!
해당 경우는 git reflog 명령어로 나오는 모든 부분 캡쳐해서 제출해주세요!
